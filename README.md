# Archisight

Программа для Windows, которая следит за одной папкой-архивом с
подпапками вида `10.07.26` и сама удаляет из неё устаревшие подпапки
(целиком, со всем содержимым), оставляя только последние N дней.
Название — Archive + Sight (архив + обзор).

---

## 1. Как понять, что программа рабочая, за 2 минуты

Тестировать GUI на Windows вручную я не мог (я работал в Linux-песочнице
без Windows и без tkinter), но **вся логика удаления вынесена в отдельный
модуль без зависимости от интерфейса** (`archisight/core.py`) и покрыта
автотестами, включая тесты ровно по твоим собственным примерам из
задания. Проверить это можно на любом ПК с Python, даже без установки
pystray/Pillow:

```
cd archisight
python -m unittest discover tests -v
```

Все 11 тестов должны пройти (`OK`). Один из тестов дословно воспроизводит
твой пример: today = 10.07.26, retention = 10 дней, папки
07.07.26…25.06.26 — и проверяет, что останутся ровно те же 10 папок,
что ты перечислил сам, а остальные удалятся.

## 3. Запуск без сборки в exe (для проверки на своём ПК)

Нужен Python 3.10+ для Windows с офсайта python.org — **при установке
галочка "tcl/tk and IDLE" должна быть включена** (она включена по
умолчанию в стандартном инсталляторе, просто не убирай её вручную).

```
pip install -r requirements.txt
python run_archisight.py
```

Откроется окно 800×600. Дальше можно сразу зайти в "Настройки" (верхнее
меню), указать папку архива и нажать "Применить сейчас" — это позволит
проверить, что программа удаляет именно то, что ты ожидаешь, ещё
до включения автозапуска.

## 4. Сборка в Archisight.exe

Всё уже настроено — просто:

```
build\build.bat
```

(двойной клик в проводнике тоже сработает). Скрипт сам поставит
зависимости, соберёт один файл `dist\Archisight.exe` со встроенной
иконкой — Python на целевом компьютере после этого не нужен, exe
самодостаточный. Можно скопировать его куда угодно (Program Files,
рабочий стол и т.д.) — просто запомни, куда положил, автозапуск
пропишет путь именно к этому расположению.

Если Windows Defender/антивирус ругается на свежесобранный exe (частое
ложное срабатывание именно на PyInstaller-сборки, особенно `--onefile`)
— это ложное срабатывание. Если мешает, в `build\build.bat` можно
поменять `--onefile` на `--onedir`: тогда вместо одного exe получится
папка `dist\Archisight\` с exe и файлами рядом — она реже триггерит
антивирусы, но её сложнее "положить куда угодно одним файлом".

## 5. Как именно считается, что удалять (важно!)

1. Смотрим ТОЛЬКО на подпапки первого уровня внутри выбранного пути
   к архиву (внутрь них программа не заходит).
2. Из них берём те, чьё имя **целиком** (не часть, а всё имя) совпадает
   с заданным в настройках форматом даты (порядок дня/месяца/года,
   разделитель, короткий/полный год).
3. Среди распознанных дат ищем самую свежую, которая не позже сегодняшней
   системной даты (это "опорная точка").
4. От опорной точки отсчитываем назад N **календарных** дней (N — из
   настройки "сколько дней хранить") — это и есть граница хранения.
5. Все распознанные папки с датой внутри этой границы — хранятся.
   Все, что старше — удаляются целиком, вместе с содержимым.
6. Пропуски дат (например, из-за выходных) НЕ ломают расчёт — просто
   для этой даты папки не было и не будет, остальные даты считаются
   так, как будто её и не было в диапазоне.
7. Всё, что не подпадает под формат даты (посторонние файлы, папки
   с другим именем/разделителем) — никогда не трогается, только
   показывается в сводке на главном экране ("Показать сводку").

### Единственный момент, где я принял решение сам — уточни, если не так

В твоём примере с пропуском (когда нет папки `01.07.26` из-за выходного)
ты перечислил среди "оставшихся" папку `27.06.26`. По формуле выше
(шаг 4) она **не проходит** — потому что граница считается по
календарным дням от опорной даты, а не по количеству папок:
при retention=10 и опорной дате 07.07.26 граница — это 28.06.26,
и `27.06.26` оказывается на день раньше границы, то есть удаляется.

Я выбрал именно такую трактовку, потому что она единственная точно
совпадает с твоим первым (более подробным) примером без пропусков —
там разница по дням и разница по количеству папок совпадали, поэтому
только пример с пропуском мог их различить. Если ты имел в виду другое
правило — например, "хранить последние N *существующих* папок, а не N
календарных дней" — скажи, это буквально одна строка кода поменять
(`compute_retention` в `archisight/core.py`), пересоберу и перепроверю
тестами за минуту.

## 6. Важные решения по интерфейсу, которые я додумал сам

- **"Сбросить"** в настройках возвращает вообще все поля к значениям
  по умолчанию (путь, время, формат даты, автозапуск выключен, дни
  хранения — 90), но, как и любые другие правки в этом окне, реально
  ничего не сохраняет и не выключает автозапуск в реестре, пока не
  нажата **"Сохранить"** — можно спокойно нажать "Сбросить" и передумать,
  закрыв окно крестиком.
- **"Применить сейчас"** — это по-настоящему удаляет файлы прямо сейчас
  (не тестовый прогон), но использует значения полей, которые сейчас
  показаны в окне настроек, даже если ты их ещё не сохранил. Отдельно
  просит подтверждение, потому что необратимо. Такой прогон не считается
  "автоматической проверкой дня" — плановая проверка по расписанию всё
  равно потом сработает как обычно.
- **Статус "Активно/Неактивно"** на главном экране смотрит на реальное
  состояние автозапуска в реестре Windows (а не просто на сохранённый
  флажок в настройках) — так показания не разъедутся, если кто-то
  вручную поменяет автозапуск в другом месте.
- **Иконка в трее** появляется только когда окно закрыто крестиком
  при включённом автозапуске (т.е. вместо полного закрытия), и исчезает
  снова, когда ты открываешь окно через неё — чтобы не висеть лишний
  раз и не путать, когда окно и так открыто.
- Короткий год (`26`) везде трактуется как `2026` (текущее столетие +
  две цифры) — для рабочего архива за последние годы этого достаточно.
- Настройки хранятся в `%APPDATA%\Archisight\config.json` и переживают
  перезагрузку компьютера.

## 7. Автозапуск

Реализован через стандартный пользовательский ключ реестра
`HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` —
это не требует прав администратора. Включается/выключается галочкой
в настройках. При запуске через автозапуск программа стартует сразу
свёрнутой в трей (флаг `--minimized`), не мешая работе с рабочим столом.
